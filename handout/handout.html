Was ist ein autonomer Agent?
01
Grundlagen / Basics

Ein autonomer Agent ist ein KI-System, das nicht nur antwortet — es handelt. Es kann Werkzeuge verwenden, Entscheidungen treffen, Aufgaben sequentiell ausführen und proaktiv auf seine Umgebung reagieren.

An autonomous agent doesn't just respond — it acts. Tools, decisions, sequences — and it can initiate without user input.

Chatbot vs. Agent

❌ Klassischer Chatbot

  • Fragt → bekommt Antwort
  • Kein Gedächtnis zwischen Sessions
  • Kein Zugriff auf externe Systeme
  • Rein reaktiv, immer on-demand

✅ Autonomer Agent

  • Empfängt Ziel → plant → führt aus
  • Persistentes Gedächtnis
  • Tools: Browser, Shell, APIs, Dateien
  • Proaktiv — arbeitet ohne User-Input

Memory-System

Agenten sind zustandslos — Memory-Dateien sind das Persistenz-Layer:

  • MEMORY.md — Langzeit-Gedächtnis (kuratiert, semantisch durchsuchbar)
  • memory/YYYY-MM-DD.md — Tagesnotizen, rohe Logs
  • SOUL.md — Persönlichkeit & Werte · USER.md — Kontext über den Menschen
  • HEARTBEAT.md — Periodische Aufgaben-Checkliste
💡 Key Insight LLM = Gehirn · Tools = Hände · Memory-Dateien = Langzeitgedächtnis. Erst zusammen entsteht echter Nutzen.

OpenClaw vs ZeroClaw
02
Vergleich / Comparison

Beide sind vollständige Agent-Runtimes — der Unterschied liegt nicht im Deployment-Ort, sondern im Technologie-Stack. Beide können lokal und in der Cloud laufen.

⚠️ Wichtig / Important OpenClaw läuft nicht nur in der Cloud, ZeroClaw nicht nur lokal. Beide können beides. Der Unterschied: Node.js vs. Rust, ClawHub vs. 65 Provider, Semantic Memory vs. SQLite.

Wann was? / Decision Guide

🐾 OpenClaw wenn…

  • JS/TS-Ökosystem bevorzugt
  • ClawHub Skills nutzen/publizieren
  • Multi-Agent mit MQTT + Subagents
  • Deep Telegram/Discord integration
  • ACP Coding-Agent Workflows

🦀 ZeroClaw wenn…

  • Minimaler Footprint wichtig
  • Hardware steuern (GPIO, STM32)
  • Maximale Provider-Auswahl (65)
  • Niche Channels (Nostr, Matrix, IRC…)
  • Migration von OpenClaw geplant

🛡 Safety Controls

  • E-Stop — Notfall-Stopp
  • OTP — Einmal-Passwort

  • max_cost_per_day ($)
  • allowed_commands — Whitelist
  • workspace_only — Sandbox

Multi-Agent — Kommunikation

📡 MQTT (OpenClaw)

  • Broker: Mosquitto (Podman/Docker)
  • Topics: hive/agent/inbox
  • Push-basiert, event-driven
  • Agents auf verschiedenen Maschinen
  • Chat-Inject in laufende Session

🔗 WebSocket Gateway (ZeroClaw)

  • Daemon öffnet HTTP + WS Server
  • ws://host:port/ws/chat
  • REST API für externe Trigger
  • Subagents via sessions_spawn()
  • ACP-Protokoll für Coding-Agents

🐉 Reales Setup: Draco (OpenClaw) + ZeroClaw (heute)

Draco läuft als OpenClaw 24/7 auf diesem Server. ZeroClaw läuft parallel mit Ollama lokal. Beide können über MQTT oder HTTP kommunizieren — unterschiedliche Channels, ein Netzwerk.

Draco (OpenClaw)  ←── MQTT ──→  ZeroClaw Demo
  Telegram/Discord/Signal         Nostr/Matrix/IRC/Bluesky

Design-Prinzipien

  • Separation of concerns — Jeder Agent hat eine klare Rolle
  • Skepticism — Agents vertrauen sich gegenseitig nicht blind
  • Async by default — Push statt Polling
  • Human-in-the-loop — Für destructive Actions immer Bestätigung

🚀 Take-Away

OpenClaw und ZeroClaw lösen dasselbe Problem auf zwei Arten. Node.js mit ClawHub und ACP vs. Rust mit 65 Providern und Hardware-Support. Beide lokal oder Cloud. Beide mit Ollama für komplette Datenprivacy.

The real shift: you give the LLM hands, memory, and a persistent identity. That's when it stops being a chatbot and starts being an agent.
